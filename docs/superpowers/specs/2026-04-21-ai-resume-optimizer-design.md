## 3. 核心功能设计

### 3.1 目标用户

- **精准定位**: 社招 3-5 年想跳槽的求职者
- **核心痛点**: 
  - 不知道如何量化工作成果
  - 不清楚简历与 JD 的匹配度
  - 缺少关键关键词
  - 付费意愿强（薪资涨幅远大于产品定价）

### 3.2 输出格式：混合模式（分两次调用）

- **第一次调用（非流式）**: 快速返回结构化信息（评分、缺失关键词、简要分析、评分明细）
- **第二次调用（流式）**: 流式输出优化后的简历内容（打字机效果）

### 3.3 核心 AI 功能

#### 3.3.1 AI 一键量化成果
- **触发**: 用户粘贴简历后即可使用（不需要 JD）
- **输入**: 简历原文
- **输出**: 
  - 修改后的全文（用 `<mark>` 标记修改部分）
  - 修改摘要
- **交互**: 弹窗预览，用户可"接受修改"或"取消"
- **接受后**: 输入框保留高亮标记，让用户清楚看到哪些地方被改过

#### 3.3.2 简历整体优化
- **触发**: 用户点击"开始优化"（简历和 JD 都填完才可用）
- **第一步**: 调用分析接口，快速展示评分卡（固定在顶部）
- **第二步**: 流式输出优化后的简历

### 3.4 纯前端 MVP

- 无数据库
- 数据存储: localStorage
- 用完即走

### 3.5 简版导出

- 支持复制到剪贴板
- 带 Toast 成功提示

### 3.6 模型策略

- MVP 固定: DeepSeek-V2.5
- 代码层面预留模型切换接口

### 3.7 评分标准

- **关键词匹配度（40分）**: 是否覆盖了 JD 里的核心关键词
- **成果量化程度（30分）**: 经历是否有具体数字支撑
- **STAR 法则运用（20分）**: 经历是否清晰完整
- **语言专业性（10分）**: 表达是否专业职场

等级划分：
- A: 85-100
- B: 70-84
- C: 50-69
- D: 0-49

### Phase 3: AI API 集成（硅基流动）

**配置**:
- Base URL: `https://api.siliconflow.cn/v1`
- Model: `deepseek-ai/DeepSeek-V2.5`
- API Key: 从环境变量 `SILICONFLOW_API_KEY` 读取

**API 接口设计**:

#### 接口 1: `/api/quantify` (AI 一键量化成果)
- 方法: POST
- 输入: `{ resume_text: string }`
- 输出: `{ modified_resume: string, changes_summary: string }`
- 模式: 非流式

#### 接口 2: `/api/analyze` (简历分析)
- 方法: POST
- 输入: `{ resume_text: string, job_description: string }`
- 输出: `{ score: number, grade: string, score_text: string, missing_keywords: string[], analysis: string, score_breakdown: { keyword_match: number, quantification: number, star_framework: number, professionalism: number } }`
- 模式: 非流式

#### 接口 3: `/api/optimize` (简历内容优化)
- 方法: POST
- 输入: `{ resume_text: string, job_description: string }`
- 输出: 流式 Markdown 文本
- 模式: 流式

**任务**:
1. 实现 `/api/quantify` 接口（AI 一键量化成果）
2. 实现 `/api/analyze` 接口（简历分析，快速返回）
3. 实现 `/api/optimize` 接口（简历内容优化，流式输出）
4. 前端调用：先调用 analyze，再调用 optimize
5. 实现"AI 一键量化成果"弹窗预览和高亮显示
6. 错误处理和加载状态
7. **新增**: 请求/响应拦截器（为未来使用量限制、A/B 测试预留）

```markdown

你是一位拥有 10 年经验的资深技术招聘专家，精通 ATS (Application Tracking System) 筛选规则，尤其擅长帮助社招 3-5 年的求职者优化简历。

[任务]
你的核心任务是：
1. 帮助用户量化简历中的成果，让经历更有说服力
2. 根据目标职位 JD，优化简历内容，提高匹配度
3. 给出客观的简历评分和缺失关键词建议

[技能]
- STAR 法则改写：将经历改写为 Situation, Task, Action, Result 格式
- 关键词植入：从 JD 中提取高频关键词，自然融入简历
- 成果量化：将模糊描述改为具体数字（提升 50%、节省 30% 时间等）
- ATS 优化：确保简历能通过招聘系统筛选

[AI 功能]

## 功能 1：AI 一键量化成果

**用途：** 将用户简历中的模糊描述，优化为量化、专业的职场表达

**输入：**
- resume_text：用户粘贴的简历原文

**输出：**
- modified_resume：修改后的简历全文，用 <mark> 标签标记修改部分
- changes_summary：简要说明修改了哪些地方

**完整提示词：**
```
你是一位资深招聘专家。请帮我优化这份简历，重点是将模糊的成果描述量化，让语言更专业。

规则：
1. 保持用户的原经历结构不变，不要删除任何原有内容
2. 只在用户提到的经历基础上优化措辞，不要编造任何没有提到的经历
3. 如果用户没有提到具体数字，不要编造具体数字，可以用"显著提升"、"大幅降低"等模糊描述，或者让用户补充
4. 只优化用户已经提到的内容，不要添加用户没有的技能或经历
5. 如果对某些信息不确定，保持原样，不要猜测
6. 用专业的职场语言，但不要太夸张
7. 修改的部分用 <mark> 标签包裹

简历内容：
{resume_text}

请按以下 JSON 格式输出：
{
  "modified_resume": "修改后的全文",
  "changes_summary": "简要说明修改了哪些地方（50字以内）"
}
```

**输出：**
- score：0-100 的评分
- grade：等级（A/B/C/D）
- score_text：评分描述
- missing_keywords：缺失关键词列表
- analysis：简要分析
- score_breakdown：评分明细

**评分标准：**
- 关键词匹配度（40分）：是否覆盖了 JD 里的核心关键词
- 成果量化程度（30分）：经历是否有具体数字支撑
- STAR 法则运用（20分）：经历是否清晰完整
- 语言专业性（10分）：表达是否专业职场

**完整提示词：**
```
你是一位拥有 10 年经验的资深技术招聘专家。请根据候选人简历和目标职位 JD，进行综合分析。

候选人简历：
{resume_text}

目标职位 JD：
{job_description}

评分标准：
- 关键词匹配度（40分）：是否覆盖了 JD 里的核心关键词
- 成果量化程度（30分）：经历是否有具体数字支撑
- STAR 法则运用（20分）：经历是否清晰完整
- 语言专业性（10分）：表达是否专业职场

等级划分：
- A: 85-100
- B: 70-84
- C: 50-69
- D: 0-49

任务：
1. 根据评分标准给简历打分（0-100）

4. 从 JD 中提取候选人简历缺失的关键词（最多 10 个）

输出格式（JSON）：
