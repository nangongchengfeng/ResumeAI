# AI 简历优化 SaaS 学习计划（硅基流动版）

## Context

你是一位全栈开发者，想通过这个项目学习：
- **AI 提示词工程** - 如何设计高质量的 Prompt
- **全栈开发流程** - 用 Next.js 从 0 到 1 构建产品
- **AI API 集成** - 如何将 AI 能力无缝集成到产品中
- **产品思维** - 从用户需求出发做产品设计

**技术选型变更**：使用 **硅基流动 (SiliconFlow)** + **DeepSeek-V2.5** 替代 Kimi K2

## 项目分析

这个 PRD 设计得非常出色，核心亮点：
1. **市场验证充分** - 16 亿美元市场规模，有成功案例佐证
2. **MVP 思路清晰** - 先做文本输入验证价值，再逐步添加功能
3. **完整闭环** - 从 Prompt 调试 → 工程化 → 产品思考

## 学习路径

### Phase 0: 硅基流动 API 准备 ⏭️ 可跳过
- **目标**：熟悉硅基流动 API，获得 API Key
- **任务**：
  1. 注册硅基流动账号，获取 API Key
  2. 用示例代码测试 DeepSeek-V2.5 流式输出
  3. 在硅基流动平台测试 Prompt 效果

### Phase 1: Prompt 工程实战（先验证核心价值）
- **目标**：在硅基流动平台调试出高质量的简历优化 Prompt
- **任务**：
  1. 准备一份真实简历文本和 JD
  2. 用 PRD 中的基础 Prompt 测试 DeepSeek-V2.5 效果
  3. 迭代优化 Prompt，加入更多约束（防幻觉、结构化输出等）
  4. 测试长文本输入（公司白皮书、技术博客等）

### Phase 2: 项目初始化与 UI 搭建
- **技术栈**：Next.js 14 + Shadcn UI + Tailwind CSS
- **任务**：
  1. 创建 Next.js 项目
  2. 配置 Shadcn UI 和 Tailwind
  3. 实现首页 Hero Section
  4. 实现核心工作台（左右两栏布局 + 按钮）
  5. 实现结果展示区（支持 Markdown）

### Phase 3: AI API 集成（硅基流动）
- **核心**：硅基流动 API + DeepSeek-V2.5 + Vercel AI SDK
- **配置**：
  - Base URL: `https://api.siliconflow.cn/v1`
  - Model: `deepseek-ai/DeepSeek-V2.5`
  - API Key: 从环境变量 `SILICONFLOW_API_KEY` 读取
- **任务**：
  1. 实现 `/api/optimize` 接口（兼容 OpenAI 格式）
  2. 集成流式输出（打字机效果）
  3. 前端调用（`useCompletion` / `useChat`）
  4. 错误处理和加载状态

### Phase 4: 产品体验优化
- **目标**：从"能用"到"好用"
- **任务**：
  1. 添加评分展示（Aha Moment）
  2. 敏感信息脱敏
  3. 防幻觉约束
  4. 排版优化（`react-markdown` + `prose`）

### Phase 5: 部署与迭代
- **任务**：
  1. 部署到 Vercel
  2. 配置环境变量（`SILICONFLOW_API_KEY`）
  3. 收集用户反馈
  4. 数据闭环设计

## 关键文件

我们将创建：
- `package.json` - 项目依赖
- `app/page.tsx` - 首页
- `app/dashboard/page.tsx` - 核心工作台
- `app/api/optimize/route.ts` - 硅基流动 AI 接口
- `lib/ai.ts` - AI SDK 配置（指向硅基流动）
- `.env.local` - 环境变量（`SILICONFLOW_API_KEY`）

## 验证方式

每个阶段完成后：
1. Phase 0: 用硅基流动示例代码测试 API 连通性
2. Phase 1: 在硅基流动平台手动测试 Prompt 效果
3. Phase 2: 打开浏览器查看 UI 布局
4. Phase 3: 实际输入简历和 JD，测试 AI 优化
5. Phase 4: 用户体验走查
6. Phase 5: 生产环境验证

## 学习重点

作为全栈开发者，特别关注：
1. **如何用 AI 辅助开发** - 跟着 PRD 中的 "Vibe Coding" 方法实践
2. **Prompt 迭代思维** - 不是一次写好，而是持续优化
3. **流式输出的用户体验** - 缓解等待焦虑
4. **产品思维 vs 技术思维** - 先验证价值，再炫技术
5. **API 兼容性** - 硅基流动兼容 OpenAI 格式，这是重要的行业标准
